Creazione modello interfaccia

Per creare una nuova interfaccia relativa ad un determinato CRF è necessario creare il file di interfaccia che deve essere inserito nella cartella CRF\ModCRF, è anche possibile personalizzare una interfaccia già esistente, senza modificarla direttamente, ma creando un file alternativo nella cartella Custom\CRF che deve essere nominato come NomePC_CRFMOD.cfg.

Il file di interfaccia contiene le istruzioni relative all'invio dei comandi al CRF oppure alla procedura interprete che si occuperà di comunicarlo al CRF. Riportiamo come esempio il file di interfaccia relativo ai CRF Ditron:

La parte iniziale commentata dove le righe sono precedute dal carattere ';' riepiloga i valori e le costanti disponibili ed il loro utilizzo all'interno del file.

La sezione Generale contiene informazioni relative all'intero modello come la dimensione delle descrizioni, se le descrizioni devono essere tutte maiuscole, se stampare i prezzi zero e se azzerare il valore negativo in chiusura (gli ultimi tre valori identificano con 0 il No e con 1 il Si).

La sezione Formattazione riporta i valori che necessitano di formattazione ed include una riga (preceduta dal carattere ';') di istruzioni su come impostare il valore da formattare.

La sezione SostituzioniTesto può essere utilizzata qualora all'interno delle descrizioni vi siano caratteri da sostituire perchè non gestiti dal CRF: le varie sostituzioni devono essere elencate progressivamente (S1, S2, Sn) e devono contenere il carattere da sostituire, il carattere '-' ed il carattere sostituto.

La sezione Sintassi contiene i comandi gestiti per ognuno dei quali si può specificare la sintassi: se la sintassi rimane vuota (ApriScontrino=) il comando sarà ignorato; la sintassi della riga deve contenere le istruzioni che quel particolare CRF, per cui stiamo facendo il modello, riceverà direttamente o tramite un programma interprete; i valori e le costanti sono riconoscibili dai qualificatori [] per i valori {} per le costanti. I comandi della sezione Sintassi, i valori, le costanti ed ogni altro elemento del file di modello vengono gestiti tramite il driver generico (OS1CRFGenerico.dll) di conseguenza non sarà possibile utilizzare una sintassi non prevista o un valore/costante non presente se non modificando tale libreria.

Una volta creato il file di modello sarà comunque necessario configurare il file utente come descritto nel capitolo Configurazione registratori di cassa; per facilitare tale funzione è stato reso disponibile il file Generico.cfg presente nella cartella Modelli\CRF che contiene tutte le possibili tipologie di collegamento del CRF ad OS1 ed il collegamento alla dll Modulo=OS1CRF_Generico. Dalla sezione CRF sono presenti le normali righe di configurazione già citate nel capitolo di cui sopra; l'unica eccezione è il comando Modello=NomeModello che solitamente conteneva il modello del CRF oppure era vuoto; utilizzando il modulo OS1CRF_Generico deve contenere il nome del file di interfaccia senza estensione (se creo l'interfaccia MioModello.crf la riga sarà Modello=MioModello).
