Configurazione eCommerce B2B

DATABASE DI TRANSITO: Indica il nome del database di transito. Fare riferimento alla procedura di Creazione database di transito.

CARTELLA DATI DI TRANSITO: Indica la cartella che contiene i dati di transito.

CARTELLA ALLEGATI ARTICOLI: Indica la cartella dove sono presenti i documenti allegati agli articoli.

CARTELLA IMMAGINI: Indica la cartella dove sono presenti le immagini degli articoli e delle categorie per il web.

DESCRIZIONE ESTESA CATEGORIE WEB: Abilita la visualizzazione della descrizione estesa delle categorie Web (casella con segno di spunta).

ALTEZZA MAX IMM. CATEGORIE: Indica l’altezza massima dell’immagine per le categorie.

LARGHEZZA MAX IMM. CATEGORIE: Indica la larghezza massima dell’immagine per le categorie.

NUMERO CATEGORIE PER PAGINA: Indica il numero di categorie visualizzate per ogni pagina web.

TIPO CALCOLO TRASPORTO: Definisce il tipo di calcolo delle spese di trasporto in fase di acquisizione ordini via Web, valori ammessi:

    • non definito,

    • In base agli imballi,

    • In base al peso,

    • In base al volume (cm cubi).

CALCOLO IMBALLI: Indica se calcolare gli imballi.

INTERFACCIA: Indica il nome dell’interfaccia web da utilizzare.

ALTEZZA MAX IMMAGINI GRANDI: Indica l’altezza massima dell’immagine visualizzata nella finestra di zoom attivata dalla scheda prodotto.

LARGHEZZA MAX IMMAGINI GRANDI: Indica la larghezza massima dell’immagine visualizzata nella finestra di zoom attivata dalla scheda prodotto.

ALTEZZA MAX IMMAGINI MEDIE: Indica l’altezza massima dell’immagine visualizzata nella scheda prodotto.

LARGHEZZA MAX IMMAGINI MEDIE: Indica la larghezza massima dell’immagine visualizzata nella scheda prodotto.

ALTEZZA MAX IMMAGINI PICCOLE: Indica l’altezza massima dell’immagine visualizzata nell’elenco prodotti.

LARGHEZZA MAX IMMAGINI PICCOLE: Indica la larghezza massima dell’immagine visualizzata nell’elenco prodotti.

Per disabilitare le immagini dei prodotti e delle categorie è sufficiente impostare le dimensioni a zero.

GESTIONE MANUALE DELLE IMMAGINI: Se spuntata, viene disabilitata la generazione automatica delle immagini in base alle dimensioni previste in configurazione.

NUMERO PRODOTTI PER PAGINA: Indica il numero di prodotti visualizzati per ogni pagina web.

TITOLO WEB: Definisce il titolo del sito Web.

INDIRIZZO PAGINA DEFAULT: Definisce l’indirizzo URL della pagina di default.

LINGUA DEFAULT INTERFACCIA WEB: Il codice lingua WEB di default.

LINGUA DEFAULT DESCRIZIONI: Indica il codice lingua OS1 di default.

NUMERO DECIMALI QUANTITÀ WEB: Indica il numero di decimali che si intende gestire per la quantità.

AMMINISTRAZIONE UTENTI SECONDARI: Definisce il metodo di amministrazione degli utenti secondari all’interno del sito Web; valori ammessi:

    • Nessuna gestione: non sono ammessi utenti secondari; il logon al sito web avverrà con codice azienda e password;

    • Gestione lato OS1: vengono gestiti in OS1 attraverso il programma di gestione Utenti.

    • Gestione lato Web: vengono gestiti dall’interno del sito ECommerce direttamente dall'utente primario (azienda e utente uguali).

CAUSALE ORDINE ABITUALE: Indica il codice causale di ordine clienti, con cui saranno importati gli ordini da Web. Se lasciata vuota sarà utilizzata la causale ordini di default presente sul cliente in OS1.

TIPO CONTROLLO RISCHIO: Definisce il tipo di controllo del rischio da utilizzare nella fase di importazione ordini da web; valori ammessi:

    • Nessun controllo: non viene effettuato nessun controllo.

    • Effettua sempre il controllo: esegue sempre il controllo.

    • Effettua il controllo in base alla causale: esegue il controllo solo se sulla causale dell’ordine è previsto.

Data consegna obbligatoria: Se attiva la data di consegna (testa/rigo in base a come gestita per il modulo delle conferme d'ordine) sarà obbligatoria; nel caso in cui il flag non sia spuntato sarà assegnata alla data di consegna la data del documento.

TIPO DI INVIO E-MAIL: Indica il tipo di invio delle email, valori ammessi:

    • MAPI (utilizzo di un client di posta elettronica per esempio OutLook)

    • SMTP (utilizzo di un server SMTP).

HOST SMTP: Definisce il nome dell'host per l’invio di email tramite SMTP.

PORT SMTP: Definisce il numero della porta per l’invio di email tramite SMTP (la porta di default per i server SMTP è la 25).

USER ID SMTP: Definisce il nome dell’user per l’invio di email tramite SMTP.

PASSWORD SMTP: Definisce la password per l’invio di email tramite SMTP.

INDIRIZZO E-MAIL: Definisce l’indirizzo email dal quale vengono inviate le comunicazioni relative all'importazione degli ordini dal web.
