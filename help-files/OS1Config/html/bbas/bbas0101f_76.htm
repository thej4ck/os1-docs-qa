Configurazione partite aperte - movimenti cash flow

La compilazione di questa tabella è obbligatoria per gli utenti che utilizzano la procedura di Contabilità ed in particolare dei moduli Gestione Partite aperte.

Codice cliente predefinito: codice cliente predefinito da utilizzare come proposta per i nuovi movimenti.

Codice fornitore predefinito: codice fornitore predefinito da utilizzare come proposta per i nuovi movimenti.

Codice sottoconto predefinito: codice sottoconto predefinito da utilizzare come proposta per i nuovi movimenti.

Data movimento predefinita: deve essere indicata la tipologia di data del movimento da proporre nei nuovi movimenti che può essere:

    • Data corrente

    • Inizio mese corrente

    • Fine mese corrente

Data scadenza predefinita: deve essere indicata la tipologia di data di scadenza del movimento da proporre nei nuovi movimenti che può essere:

    • Data corrente

    • Inizio mese corrente

    • Fine mese corrente

    • Data movimento

    • Data movimento + giorni, in questo caso alla data del movimento vengono sommati i giorni indicati nel campo successivo “Numero giorni per scadenza”.

Numero giorni per scadenza: Numero di giorni da sommare alla data del movimento; valido se il campo precedente è valorizzato con 'Data movimento + giorni'.
