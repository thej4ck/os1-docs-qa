Aggiornamento automatico semafori

Il programma AggiornaCreditShield ha la funzione di effettuare l’aggiornamento automatico dei semafori di OS1, con la possibilità di schedularne il lancio tramite l’utilità di pianificazione presente nei sistemi Windows.

Il programma può essere lanciato in tre diverse modalità:

    1. Senza parametri: in questo caso utilizza la configurazione standard di registro per ottenere i parametri di accesso al database di logon.
    2. Con parametro che corrisponde al nome di una multi-configurazione: in questo caso utilizza i valori della configurazione indicata per ottenere i parametri di accesso al database di logon.
    3. Con parametro che corrisponde ad un file: in questo caso utilizza i valori contenuti nel file per recuperare i parametri di accesso al database di logon. La struttura del file è la seguente:

[Connessione]

Server=Nome o IP del server

DataBase=OSILogon

UserName=sa

Password=MiaPassword

 

Indipendentemente dalla modalità, una volta individuato il database di logon, il programma analizza tutte le Ditte configurate; per ogni ditta che ha impostate le credenziali di accesso ai servizi CreditShield e il flag di aggiornamento automatico, il programma procede all’aggiornamento dei valori semaforici.

 

Esiste inoltre la possibilità di utilizzare un file di configurazione “AggiornaCreditShield.ini”, posto nella stessa cartella del programma, dove si possono impostare i parametri per l’invio di notifiche via email. Con l’installazione viene reso disponibile un file di esempio (AggiornaCreditShield.ini.sample) il cui contenuto è il seguente:

[Opzioni]

; Se diverso da 0 abilita l'invio di email

InviaEmail=0

; Se diverso da 0 invia email solo in caso di errori

EmailSoloSuErrore=0

 

[SMTP]

; Impostare TestInvio = 1 per fare prove di invio email; nessun aggiornamento verrà eseguito

TestInvio=0

; Specificare l'indirizzo dell'host e l'eventuale port (default = 25)

Host=MioHostSMTP

Port=MiaPortaSMTP

; Specificare il tipo di sicurezza: (vuoto = nessuna, SSL, TLS)

Sicurezza=

; Se il provider richiede autenticazione specificare Utente e Password

UtenteSMTP=

Password=

; Indirizzo email mittente

Mittente=

; Indirizzo email destinatario (se sono più di uno separarli con punto e virgola ";")

Destinatario=

; Specificare l'oggetto della mail inserendo anche il simbolo %s per consentire

;di integrare il messaggio prodotto dall'applicazione. Valore predefinito:

;Oggetto=Aggiornamento CreditShield: %s

Da notare che se viene impostata la voce TestInvio=1, il programma non effettua alcun aggiornamento, ma vengono utilizzati i parametri SMTP per inviare un’email di prova.

È inoltre possibile specificare una parte dell’oggetto del messaggio che si va a comporre con la parte generata dal programma.
