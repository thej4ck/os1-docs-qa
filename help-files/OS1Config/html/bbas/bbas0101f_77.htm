Configurazione gestione matricole

Gestione matricole: Attiva o disattiva la gestione matricole.

Data inizio gestione: È la data iniziale di utilizzo delle matricole legate ai prodotti; movimenti o documenti antecedenti questa data non richiederanno o mostreranno la matricola anche se gestita sul prodotto.

Gestione scadenza matricole: Indica se controllare l'eventuale data di scadenza presente sulla matricola per impedirne la movimentazione se antecedente alla data del documento/movimento (casella con segno di spunta).

Causale Ddt mov. matricole scadute: Indicare la causale Ddt di vendita utilizzata per scaricare, tramite questo tipo di documento, matricole scadute.

Causale fatture mov. matricole scadute: Indicare la causale fattura di vendita utilizzata per scaricare, tramite questo tipo di documento, matricole scadute.

Caus. magazzino mov. matricole scadute: Indicare la causale di magazzino utilizzata per scaricare, tramite la prima nota di magazzino, matricole scadute.

Data base per creazione matricole: Viene utilizzato per assegnare la data di creazione della matricola ed eventualmente la conseguente data di scadenza. Può assumere i valori "Data di sistema" o "Data registrazione"; se quest'ultima data non fosse reperibile sarà comunque utilizzata la data di sistema.

Tipo gestione univocità matricole: Definisce l'univocità della matricola:

    • Matricola univoca per articolo: il codice matricola sarà univoco per prodotto, sarà possibile attribuire la stessa matricola a prodotti diversi.

    • Matricola univoca per classe: il codice matricola sarà univoco in base alla classe, se questa viene definita univoca non sarà possibile attribuire la stessa matricola altrimenti il controllo sarà per articolo.

    • Matricola univoca aziendale: il codice matricola sarà univoco per tutta l'azienda, non sarà possibile avere matricole uguali a nessun livello.

Tipo barcode: Definisce il tipo di barcode preferenziale per le matricole; utilizzato per proporre il tipo in fase di stampa etichette.

Consenti scarico matricole non caricate: Se spuntato, consente l'utilizzo di matricole non caricate e/o la generazione di matricole nuove in fase di scarico.

TIPO CONTROLLO PRESENZA MATRICOLE: definisce il tipo di controllo da applicare in fase di salvataggio dei documenti/movimenti; il controllo vincolante non consente il salvataggio se tutte le matricole non sono state assegnate, il controllo non vincolante comunque segnala l'incoerenza ma consente il salvataggio del documento/movimento. Il parametro non è applicato all'inventario fisico, ai versamenti della produzione ed ai rientri del c/to lavoro dove è sempre vincolante in relazione alla fase.

Tipo assegnazione matricole produzione: Il parametro è attivo solo se presente il modulo Produzione. Se la casella è spuntata le matricole dovranno essere assegnate sul rientro/versamento dell'ultima fase; se presenti matricole prenotate in RDP saranno assegnate automaticamente. Per quanto riguarda le fasi precedenti non sarà possibile effettuare nessuna assegnazione matricole. Se la casella non è spuntata le matricole dovranno essere assegnate sul rientro/versamento della prima fase; se presenti matricole prenotate in RDP saranno assegnate automaticamente. Per quanto riguarda le fasi successive sarà effettuata un'assegnazione automatica dalla fase precedente.
