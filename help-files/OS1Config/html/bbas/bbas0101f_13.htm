Configurazione conferme d'ordine

La compilazione di questa tabella è richiesta agli utenti che utilizzano il modulo Gestione Ordini Clienti. Ciascun check box parametrizza l'abilitazione della richiesta della relativa data nei programmi di emissione ordini.

DATA RICHIESTA DI TESTA: Abilita, in gestione ordini clienti, l’inserimento della data di richiesta consegna di testa (casella con segno di spunta).

DATA PRODUZIONE DI TESTA: Abilita, in gestione ordini clienti, l’inserimento della data di produzione di testa (casella con segno di spunta).

DATA CONSEGNA DI TESTA: Abilita, in gestione ordini clienti, l’inserimento della data di prevista consegna di testa (casella con segno di spunta).

DATA RICHIESTA DI RIGO: Abilita, in gestione ordini clienti, l’inserimento della data di richiesta consegna di rigo (casella con segno di spunta).

DATA PRODUZIONE DI RIGO: Abilita, in gestione ordini clienti, l’inserimento della data di produzione di rigo (casella con segno di spunta).

DATA CONSEGNA DI RIGO: Abilita, in gestione ordini clienti, l’inserimento della data di prevista consegna di rigo (casella con segno di spunta).

GESTIONE MISURE PRODOTTI: Definisce se all'interno della gestione ordini cliente e' attivo l'inserimento delle misure del prodotto (lunghezza, larghezza, altezza) al fine di calcolare la quantità.

IMPORTA DESCRIZIONE ESTESA: Se attivo in fase di emissione ordine per i prodotti la cui descrizione estesa non è fissa viene riportata la descrizione estesa del prodotto nel primo campo di annotazioni disponibile sul rigo in modo tale da poterlo modificare.

TIPO EVASIONE OFFERTA: Utilizzato nel programma di evasione offerte indica il criterio di evasione preferenziale che si intende utilizzare, attivando la pagina opportuna; valori ammessi: per n. offerta, per prodotto, per data di consegna.

TIPO EVASIONE ORDINE: Utilizzato nel programma di evasione ordini, per evadere gli ordini aperti, indica il criterio di evasione preferenziale che si intende utilizzare, attivando la pagina opportuna; valori ammessi: per n. ordine, per prodotto, per data di consegna, per n. ordine (ordinata data/num decrescente).

RIF. ORDINE SU DOCUMENTI: Definisce se si desidera generare un rigo descrittivo, in fase di evasione ordine cliente, che indica il documento evaso; valori ammessi:

    • Nessun riferimento: non genera il rigo descrittivo.

    • Riferimento interno: genera rigo descrittivo contenente causale, numero e data dell’ordine..

    • Riferimento esterno: genera rigo descrittivo contenente il riferimento inserito direttamente sulla testa dell’ordine nel campo “Vs. ordine”.

    • Entrambi i riferimenti: indica sia il riferimento interno che quello esterno.

DESCRIZIONE PER DOCUMENTO: Breve descrizione, utilizzata in fase di evasione ordine cliente nel caso in cui si desideri generare il riferimento, che intesta il riferimento stesso.

TIPO CONTROLLO QUANTITÀ EVASA: Utilizzato in fase di evasione ordine cliente delinea il tipo di controllo da effettuare ogni qual volta si selezioni una quantità superiore a quella definita sul rigo del documento, valori ammessi:

    • Nessuno: non effettua nessun controllo.

    • Controllo non vincolante: avvisa con un messaggio ma lascia proseguire l’operazione.

    • Controllo vincolante: avvisa con un messaggio e impedisce l’operazione.

TIPO CONTROLLO DISPONIBILITÀ: Indica il tipo di controllo da effettuare in fase di emissione ordine cliente sulle disponibilità di magazzino. Tale controllo, se attivo, può essere vincolante o meno e può essere attivato solo sul magazzino indicato sul rigo del movimento oppure su tutti i magazzini di proprietà dell’azienda.

Richiesta criteri di selezione in evasione: Definisce se richiedere, in fase di evasione, dei criteri di selezioni supplementari per la selezione dei documenti da evadere (casella con segno di spunta).

Evasione ordini di altro cliente: Consente di definire se è possibile evadere ordini di altri clienti, o meno; valori possibili:

    • Non consentito: il campo cliente in fase di selezione ordini non può essere modificato.

    • Cliente obbligatorio: il campo cliente in fase di selezione ordini può essere modificato sostituendolo con un altro codice valido.

    • Cliente facoltativo: il campo cliente in fase di selezione ordini può essere modificato e lasciato vuoto.

Gestione ordini sospesi: Attiva la gestione degli ordini sospesi (casella con segno di spunta); la gestione prevede per ogni ordine lo stato di sospensione che esclude l'ordine dall'evasione.

Tipo blocco ordini accettati: Utilizzato in caso di gestione ordini sospesi per indicare quali operazioni non sono consentite una volta che l'ordine viene accettato; valori possibili:

    • Nessun blocco

    • Blocca variazione

    • Blocca cancellazione

    • Blocca variazione e cancellazione

Gestione impegno ordini sospesi: Se la casella è spuntata l'ordine sospeso movimenta comunque l'impegnato da clienti in base alla causale con cui è stato inserito, altrimenti (casella vuota) l'ordine sospeso, qualsiasi causale utilizzi, non movimenta l'impegnato da cliente.

Controllo prezzo in ordine: Se attivo in fase di inserimento ordine viene segnalato se il rigo ha prezzo uguale a zero.
