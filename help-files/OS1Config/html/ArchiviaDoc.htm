Configurazione esterna archiviazione

Per il corretto funzionamento di ArchiviOK in modalità integrata con OS1 in versioni antecedenti alla 3.1 oppure dalla versione 3.1 senza utilizzare l’interrogazione diretta da OS1 (utilizzo di XAOKPlugIn), i file SmartDb.dll e SmartUtils.dll presenti nella sottocartella Program di installazione di ArchiviOK, devono essere copiati nella cartella principale di OS1.

 

Per configurare l'archiviazione è necessario creare nella cartella Custom una cartella Archiviazione e copiare dalla cartella Modelli\Archiviazione il file per l'utilizzo della procedura di archiviazione.

Il file copiato deve essere rinominato con nome macchina seguito dal suffisso '_ARCHDOC' e deve avere estensione cfg. Ad esempio: MIOPC_ARCHDOC.cfg

 

Struttura dei file di configurazione

Il file contiene parametri specifici e delle righe di commento, precedute dal carattere ';' che spiegano come effettuare gli eventuali settaggi.

Attenzione, se il parametro Scanner, relativo all'acquisizione di documenti esterni, ha valore 0 (zero), l'utente utilizzatore non avrà la facoltà di archiviare documenti esterni.

Riportiamo un esempio:

[Config]

;Determina l'abilitazione all'acquisizione dei documenti: 0=No 1=Solo scanner 2=Solo PDF 3=Sia scanner che PDF

Scanner=3

;Indica se, in caso di acquisizione da scanner, il documento viene acquisito subito o successivamente

;0 = acquisizione immediata; 1 = acquisizione differita; 2 = richiedere ogni volta

;In caso di acquisizione differita vengono archiviate solo le chiavi ed effettuata la stampa di un barcode

Differita=2

;Path della procedura di archiviazione documentale

Programma=C:\Programmi\ArchiviOK\Program\ArchiviOK.exe

;Utilizza le funzionalità della XAOKPlugIn che estendono/sostituiscono le funzionalità di ArchiviOK: 1=si 0=No

UsaXAOKPlugIn=1 [Se il parametro non esiste viene inteso come impostato a 1, nel caso sia impostato a 1 non è necessario copiare le dll sopra descritte]

 

 

Nella cartella Modelli\Archiviazione\ArchiviOK sono presenti i files di configurazione relativi alle tipologie di documento gestite; di seguito riportiamo un esempio del file di configurazione relativo alle fatture di vendita.

 

FatturaV

[Dati]

Select=SELECT F.IdCliente, C.RagioneSociale, F.Anno, F.Serie, F.Numero, F.IdCausale, F.DataDocumento FROM FattureCliTeste F LEFT JOIN Clienti C ON C.IdConto = F.IdCliente WHERE F.KFattCliT = :Chiave

NomeFile=FTV[Anno]-[Serie]-[Numero].pdf

Titolo=FT VEN. N. [Anno]-[Serie]-[Numero]

;Specificare in caso si voglia effettuare l'acquisizione differita. Deve essere di 2 caratteri numerici.

;Se attiva l'acquisizione differita il titolo non viene creato come indicato nella voce "Titolo", ma in modo automatico utilizzando un barcode EAN13.

;TipoDocumento=01

 

[ArchiviOK]

GRUPPO=%%Archivio%Vendite%Fatture%

MODELLO=Fattura

 

[Chiavi]

; Per valori costanti far precedere il valore dal carattere %

TipoAnag=%CL

CodiceAnag=IdCliente

RagSoc=RagioneSociale

Causale=IdCausale

DataDoc=DataDocumento

Numero=Numero

Anno=Anno

Serie=Serie

 

Il file è diviso in sezioni il cui nome è racchiuso tra parentesi quadre.

Il primo gruppo [Dati] contiene la sintassi SQL (Select) per l'estrazione dei dati, la struttura del nome del file (NomeFile) con cui saranno salvati i documenti archiviati ed il formato del titolo (Titolo) per il documento archiviato ed il tipo documento (TipoDocumento) nel caso si utilizzi l'archiviazione differita. I nomi dei campi riportati tra parentesi quadre fanno riferimento ai campi presenti nell'istruzione SQL e riporteranno dove inseriti il valore.

Il gruppo [ArchiviOK] è relativo alla procedura di archiviazione e deve rimanere fisso.

Il gruppo [Chiavi] contiene l'indicazione delle possibili chiavi di ricerca del documento all'interno della procedura di archiviazione con il relativo campo o valore costante (preceduto dal carattere %).
