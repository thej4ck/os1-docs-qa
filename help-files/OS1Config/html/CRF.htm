Configurazione esterna registratori di cassa

Modelli di stampanti fiscali RT compatibili XML7:

Ditron Zip (MicroPOS WinEcrCom v.3.1.1)

Custom con protocollo XOn/XOff

Epson - EpsonFpMate v.5.6.0.33 (EpsonFp2Control 4.3.29)

Olivetti con ElaExecute v.1.17.27_00

Coristech con protocollo XOn/XOff

 

Per configurare il collegamento con una stampante fiscale, è necessario creare nella cartella Custom una cartella CRF e copiare dalla cartella Modelli\CRF il file relativo al modello utilizzato.

Il file copiato deve essere rinominato con nome macchina seguito dal suffisso '_CRF' e deve avere estensione cfg. Ad esempio: MIOPC_CRF.cfg

E' possibile rinominare il file con nome macchina seguito dal carattere '_', seguito dal nome utente e dal suffisso '_CRF', sempre  con estensione cfg (ad esempio MIOPC_UTENTEA_CRF.cfg); questo meccanismo si rivela utile se più utenti collegati in remoto su un unico server hanno per esempio la cassa collegata localmente e di conseguenza ogni utente dovrà avere le proprie impostazioni.

Esiste la possibilità di simulare la presenza del registratore di cassa senza stampare nessun tipo di scontrino; per utilizzare questa impostazione è necessario utilizzare i file di configurazione: NoCRF.cfg.

Esiste anche la possibilità per un operatore di poter stampare scontrini su più casse anche diverse fra loro; se nasce questa esigenza è sufficiente creare un file CA@NNNNN_CRF.cfg (dove NNNNN è il numero della cassa preceduto da zeri 00001) nella cartella Custom\CRF utilizzando il file della Modelli\CRF relativo al modello utilizzato, questo perchè il file DEVE avere la stessa struttura del file NOMEPC_CRF.CFG (che deve comunque essere presente).

 

Struttura dei file di configurazione

Ogni file ha parametri specifici e contiene anche delle righe di commento, precedute dal carattere ';' che spiegano il significato delle righe presenti.

Alcune opzioni hanno lo stesso significato:

  • Tipo comunicazione: 0=Seriale 1=Tramite file di appoggio

  • Parametri: in caso di collegamento seriale indicare la porta (es. COM1:); altrimenti indicare il percorso ed il nome del file da generare (es. C:\Scontrini\Scontrino.txt); se viene omessa l'estensione del file (es. C:\Scontrini\Scontrino.) i file saranno creati automaticamente con estensione numerica progressiva (.1, .2, .3, ecc).

  • Modulo: nome della bpl di OS1 (es. OS1CRF_Ditron)

  • NumDecimali: indicare il numero di decimali da utilizzare in OS1 per mantenere la coerenza degli importi; il valore -1 indica di utilizzare i decimali della divisa di OS1

Solitamente una funzionalità può essere attiva (valore=1) o disattiva (valore=0); alcune funzionalità hanno anche altri valori specifici; in ogni caso devono essere impostate in base a cosa il CRF può gestire.

Maggiori informazioni alla pagina Composizione file di configurazione CFG.

 

Configurazione CRF XOn/XOff collegato in LAN

Per effettuare la stampa dello scontrino via ethernet con CRF XOn/XOff compatibili (che possono essere impostati su una LAN) è necessario inserire nel file cfg, nella sezione [Config] le seguenti righe:

 

TipoComunicazione=1 Indica di salvare su file
Parametri=C:\Scontrini\Scontrino.txt

Percorso e nome del file se modificato è necessario modificarlo anche nel file Invia.bat
PathPgmExe=C:\Scontrini\Invia.bat

Percorso e nome del file da eseguire dopo il salvataggio dello scontrino

 

Il programma tcpcat è presente nella cartella servizi di OS1; il file Invia.bat deve risiedere nella cartella dove viene salvato il file scontrino deve contenere la seguente riga:

C:\OSItalia\OS1\Servizi\tcpcat 192.168.1.179 9100 C:\Scontrini\Scontrino.txt (dove 192.168.1.179 è l’IP del CRF e 9100 è la porta default)

Il programma tcpcat può avere anche altri due parametri aggiuntivi opzionali nel caso durante la comunicazione ci fosse perdita di dati, le opzioni complete sono le seguenti:

tcpcat indirizzoIP Port "PathFile" [MaxSizeScontrino(999999)] [LinesToSend(5)]]

Il MaxSizeScontrino indica in Byte un limite di dimensione e, se il file che si sta trattando supera tale size è necessario indicare il numero di righe (LinesToSend) in cui lo si vuole spezzare; è da tenere presente che ad ogni divisione del file e quindi invio dello stesso al CRF corrisponde il tempo di attesa di un secondo.
