Configurazione esterna invio massivo eMail/Fax

 

Allo stato attuale i documenti che dispongono del servizio di invio massivo sono:

    • Fatture clienti (tipo documento 1)

    • Ddt clienti (tipo documento 0)

    • Ordini clienti (tipo documento 2)

    • Offerte clienti (tipo documento 4)

    • Packing list (tipo documento 3)

    • Ordini fornitori (tipo documento 12)

    • Ordini terzisti (c/lavoro) (tipo documento 22)

    • Ddt terzisti (c/lavoro) (tipo documento 20)

    • Parcelle proforma (tipo documento 101)

    • Solleciti clienti (tipo documento 300)

    • Estratto conto clienti/fornitori (tipo documento 301)

 

Il tipo documento, indicato tra parentesi, è importante poiché per questo modulo il file di configurazione eventualmente da parametrizzare per l’invio è denominato TipoDoc=x.cfg dove x è il tipo documento. La scelta è stata fatta al fine di rendere più agevole l’eventuale implementazione di altri documenti evitando di aggiungere ulteriori tabelle di configurazione. I files di configurazione relativi alle tipologie di documento gestite sono presenti nella cartella Modelli\EMailFax.

 

TipoDoc=1

; Fattura di vendita

[Dati]

Select=SELECT F.IdCliente, C.RagioneSociale, F.Anno, F.Serie, F.Numero, F.IdCausale, F.DataDocumento, P.DsSerie, F.IdDestinazione, F.IdDestinazioneAmm FROM FattureCliTeste F INNER JOIN Clienti C ON C.IdConto = F.IdCliente INNER JOIN ProtIva P ON P.Anno = F.Anno AND P.Serie = F.Serie WHERE F.KFattCliT = :Chiave

NomeFile=FTV[Anno]-[Serie]-[Numero].pdf

 

[MailMerge]

; Per valori costanti far precedere il valore dal carattere %

FlTipoDocumento=%1

TipoAnag=%CL

CodiceAnag=IdCliente

IdDestinazione=IdDestinazione

IdDestinazioneAmm=IdDestinazioneAmm

RagioneSociale=RagioneSociale

Anno=Anno

Serie=Serie

Numero=Numero

Data=DataDocumento

Causale=IdCausale

DsSerie=DsSerie

 

Il file è diviso in sezioni il cui nome è racchiuso tra parentesi quadre.

Il primo gruppo [Dati] contiene la sintassi SQL (Select) per l'estrazione dei dati, la struttura del nome del file (NomeFile) con cui saranno salvati i documenti inviati. I nomi dei campi riportati tra parentesi quadre fanno riferimento ai campi presenti nell'istruzione SQL e riporteranno dove inseriti il valore.

Il gruppo [MailMerge] contiene l'indicazione dei campi del documento con il relativo campo o valore costante (preceduto dal carattere %); dette voci è possibile riportarle all'interno della procedura OS1, ad esempio inserendo il nome del campo tra parentesi quadre ([RagioneSociale] nel testo del messaggio da allegare alla eMail. E' possibile aggiungere anche altri campi purché presenti nella sintassi SQL.
