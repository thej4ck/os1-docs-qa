Bonifici / Ritiro effetti - Home banking

Il programma crea il “file pagamenti” che contiene l’elenco informatico dei pagamenti relativi ad una determinata distinta di pagamento definitiva, secondo le specifiche del tracciato record utilizzato dall’ABI (Associazione Bancaria Italiana), oppure un file di testo o XML secondo le specifiche SEPA.

Tale file, a cui deve essere assegnato un nome non più lungo di 8 caratteri e privo di punti e caratteri divisori, può successivamente essere trasmesso alla banca tramite un programma di home banking, oppure può essere generato su un floppy disk per la consegna, unitamente alla distinta, alla banca.

 

ESERCIZIO: inserire l'anno relativo alla distinta da generare. Viene proposto il numero dell’esercizio in corso.

NUMERO DISTINTA: inserire il numero progressivo della distinta di pagamento di cui si vuole generare il file Ri.Ba. Normalmente viene presentato il numero dell’ultima distinta stampata.

DATA CREAZIONE: viene proposta la data del giorno, modificabile, che verrà memorizzata sul file.

Tipo esito ordinante: Il campo consente di scegliere come gestire l'esito ordinante nel flusso XML generato. Valori disponibili:

      • senza esito all'ordinante (TRF); parametro di default

      • con Esito all'ordinante (TRA)

ADDEBITO CUMULATIVO: Il campo consente di scegliere se gestire l'addebito cumulativo (campo BtchBookg) nel flusso XML generato. Valori disponibili:

      • Non gestito (NO - False). Parametro di default.

      • Gestito (SI - True).

      • Non impostare (il campo non viene inserito nel file XML).

FILE DESTINAZIONE: inserire un nome file valido; il pulsante accanto al campo consente di scegliere un determinato percorso oltre al nome del file.

 

Gli utenti che intendono generare il file per acquisirlo tramite home banking, devono anteporre al nome del file di destinazione il percorso per indicare la directory di destinazione in cui deve essere salvato il file.
