Classi lotti

La classe del lotto definisce la tipologia di creazione del codice del lotto stesso; viene associata all’articolo tramite l’anagrafica articoli. Dalla procedure di manutenzione è possibile effettuare la stampa delle classi utilizzando i tasti funzione della toolbar.

 

Per ogni classe è presente un codice, una descrizione; nelle procedure dove gestito il codice lotto sarà generato in automatico in base alla struttura definita sulla classe stessa.

La struttura del codice lotto può essere composta da una serie di valori così riepilogati:

    • Nessuno: da utilizzare per inserire delle parti fisse nel codice del lotto tramite il campo Valore.

    • Esercizio: utilizza l'esercizio attuale al momento della creazione.

    • Anno: calcolato in base alla data di sistema al momento della creazione.

    • Semestre: calcolato in base alla data di sistema al momento della creazione.

    • Quadrimestre: calcolato in base alla data di sistema al momento della creazione.

    • Trimestre: calcolato in base alla data di sistema al momento della creazione.

    • Mese: calcolato in base alla data di sistema al momento della creazione.

    • Settimana: calcolato in base alla data di sistema al momento della creazione.

    • Giorno dell'anno: calcolato in base alla data di sistema al momento della creazione.

    • Giorno del mese: calcolato in base alla data di sistema al momento della creazione.

    • Progressivo: utilizza un progressivo per esercizio; il valore viene memorizzato nella tabella protocolli lotti (deve essere l’ultimo campo del codice).

    • Progressivo periodico: utilizza un progressivo calcolato in base alla combinazione dei periodi impiegati per la formazione del codice (deve essere l’ultimo campo del codice).
